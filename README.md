# Manong Arena

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![GitHub Release](https://img.shields.io/github/v/release/qq1171910065/manong-arena)](https://github.com/qq1171910065/manong-arena/releases)

**Manong Arena**（产品代号 `arena`）是一款开源的 **AI 角色养成** 与规则化社交场景 **桌面客户端**（MIT）。当前版本 **v0.1.1**。

> 养成你的 AI 角色，在讨论、博弈等社交范式中检验人设与提示词工程。

- **GitHub**：https://github.com/qq1171910065/manong-arena  
- **下载安装包**：见 [Releases](https://github.com/qq1171910065/manong-arena/releases)（Windows `.exe` / macOS `.dmg`）  
- **开源说明 / 发版流程**：[`docs/OPEN_SOURCE.md`](docs/OPEN_SOURCE.md)

本仓库**仅包含客户端**。登录、模型网关、钱包与发版接口需对接你自建的 [Platform](https://github.com/czmanong/manong-arena#账号与模型) 后端（默认不在安装包内写死私有地址，请在应用设置中配置 Platform URL）。

你可以创建多个拥有独立人设、口吻与策略倾向的 AI 角色，安排他们进入狼人杀、圆桌讨论等玩法，观看他们在规则约束下推理、交锋、投票，并留下可复盘的对局记录。你还可以创建**专属的 AI 分身**——平时由 AI 替你上场，必要时亲自接管发言。

完整产品愿景见 [`docs/Agent_Arena_产品说明书_V1.1.md`](docs/Agent_Arena_产品说明书_V1.1.md)（含尚未落地的规划内容；文档中部分名称仍写作 Agent Arena）。

## 这是什么

Manong Arena 不是普通在线多人游戏，也不是单纯的 AI 聊天室。它更像一个 **AI 角色养成 + 社交范式实验场**：

- **角色是长期资产**：每个 AI 都有名字、立绘、性格、说话方式、策略偏好，可跨多次场景演练持续培养。
- **场景是规则容器**：纯讨论、规则化社交博弈（如狼人杀）等，是同一套「社交范式 + 提示词工程」的不同实例。
- **你是导演，也可下场**：你负责选角色、选场景、发起演练；默认由 AI 在规则内行动。若本局包含**你的 AI 分身**，可在需要时**亲自接管**该席位发言。

## 核心玩法循环

```text
首次启动 → 下载素材 → 初始化内置角色与玩法
    ↓
创建你的 AI 分身（昵称、说话风格）
    ↓
创建 / 培养其他 AI 角色
    ↓
为角色学习玩法规则（可选考试）
    ↓
选择玩法场景，挑选参与角色（可包含你的分身）
    ↓
创建对局 → 进入对局房间
    ↓
AI 按阶段发言、投票、行动（分身默认也由 AI 操控）
    ↓
必要时：真人接管分身发言 → 提交后继续由 AI 推进其余流程
    ↓
对局结束 → 复盘、赛后成长、重新开局
```

## AI 角色

每个 AI 角色都可以长期保存，并拥有自己的档案：

| 维度 | 说明 |
|------|------|
| 形象 | 头像、立绘、表情 |
| 人设 | 性格标签、背景、说话风格、常用口头禅 |
| 策略 | 激进 / 谨慎、逻辑 / 共情、带节奏 / 跟票等倾向 |
| 模型 | 可绑定不同大模型，同一玩法里多个 AI 可以「各怀绝技」 |
| 玩法技能 | 是否已学习某玩法、是否通过考试、赛后沉淀的理解与策略 |

角色支持导入 / 导出（`*.arena-character.zip`），方便迁移与分享。内置 starter 角色会在首次初始化时安装。

**赛后成长**：对局结束后，角色可参与赛后复盘，根据表现调整行为准则，让下一局更像「同一个角色在进步」。

### 你的 AI 分身

入门初始化完成后，会引导你创建**全局唯一**的 AI 分身：

- 绑定你的昵称与说话风格，作为「你在牌桌上的 AI 替身」
- 创建对局时与其他角色一样选入席位；未选中分身时，本局不可真人介入
- 分身档案可在角色库中继续编辑（形象、原则、模型等）

平时由 AI 替你完成发言；**只有当你的分身在本局参战时**，对局房间才会出现「真人参与」入口。

## 玩法介绍

### 狼人杀 · 经典身份推理

当前最完整的规则玩法。6–18 人局，推荐 10 人，单局约 40 分钟。

**阵营与目标**

- **好人阵营**：通过发言与投票找出狼人；狼人全部出局则好人胜。
- **狼人阵营**：隐藏身份，夜晚刀人、白天带节奏；可选 **屠边**（杀光平民或神职）或 **屠城**（杀光所有好人）获胜。

**一局怎么进行**

1. **警上发言 & 警长投票** — 首夜行动前的竞选；预言家此阶段尚未查验，跳身份只能谈警徽流。
2. **夜晚行动** — 守卫守护 → 狼人刀人 → 预言家查验 → 女巫用药，依次结算。
3. **白天发言** — 公布死亡信息，按席轮流发言，警长拥有 1.5 票归票权。
4. **放逐投票** — 投票淘汰可疑玩家，进入下一轮或结算胜负。

**常见身份**

| 阵营 | 身份 | 要点 |
|------|------|------|
| 狼 | 狼人 | 夜晚共同刀人，白天伪装好人 |
| 好 | 村民 | 无技能，靠逻辑与票型找狼 |
| 好 | 预言家 | 每晚查验阵营，结果仅自己可见 |
| 好 | 女巫 | 一瓶解药、一瓶毒药，各用一次 |
| 好 | 守卫 | 每晚守护一人，不能连续两晚守同一人 |
| 好 | 猎人 | 被刀或放逐时可开枪（被毒除外） |

**人数与板子**：6 人起预女猎守等标准神职随人数递增；10 人及以上可启用扩展身份（骑士、白痴、守墓人、狼王、狼美人、白狼王等）。

**规则扩展**（创建对局时可选）：放逐遗言、夜间遗言、毒杀遗言；屠边 / 屠城模式。

**身份分配**：支持随机、手动、半随机（锁定部分身份后其余随机），方便测试「某个角色当狼人时会怎样」。

### 圆桌讨论 · 纯议题碰撞

2–12 人，推荐 4 人，单局约 20 分钟。**无隐藏身份、无投票、无胜负、无产物输出、无经验教训沉淀**——只保留最基础的议题讨论。

开局前设定 **讨论议题** 与 **发言轮数**（默认 3 轮）。可选 AI 主持人引导轮次，可选解说中立概括。

---

### 头脑风暴 · 讨论 + 产物（独立玩法）

头脑风暴与圆桌**拆分为独立玩法**，第一阶段提供两类：

| 玩法 | 产物 |
|------|------|
| **头脑风暴 · 玩法设计** | 规则草案、阶段流程、开局配置要点 |
| **头脑风暴 · 角色塑造** | 人设原则、口癖、行为禁忌清单 |

配置项：议题、轮数、**设计焦点**。讨论结束后由系统归纳 **artifact**（可在角色「场景记录」的「讨论产物」tab 查看）。**不产生经验教训沉淀**。

---

### 谁是卧底

4–10 人轻量社交博弈：持有相近词语，轮流描述、投票出局。已作为独立玩法接入（第一阶段简化引擎：描述 + 投票）。

---

### 玩法导入导出与素材包

- **内置玩法不可删除**；修改提示词 / 规则后可 **导出配置包**（`*.arena-mode.json`，格式 `game-mode-pack-v2`），再 **导入同名玩法** 以更新。
- 导出包含：玩法元数据、场景文档、**全部提示词包**、**素材 manifest**（封面 `mode-cover`、对局横幅 `match-banner` 相对路径）。
- **自建玩法**（`user-mode-*`）可正常删除；导入他人包时会分配新 id。

在「玩法场景」中也可 **新建玩法**（基于圆桌模板），导出 / 导入 `*.arena-mode.json` 分享配置。

## 对局体验

**对局房间**以消息气泡、角色头像、当前阶段、投票区为核心，更接近「AI 同场聊天 + 规则推进」，而非传统战斗界面。

**默认视角为玩家视角**（不持久化到设置）：隐藏真实身份，适合沉浸式观战；需要时可切换**上帝视角**查看全场身份与 AI 思考过程。**注意**：本局一旦开启上帝视角，将无法再次「替代分身加入」——避免全知信息与真人操控混用。

局内你可以：

- 观看公开发言、投票结果与阶段变化（AI 会追踪场上公开跳身份，悍跳后前后呼应；阅读他人发言时区分核心论点与随口提及，内部做更深站边推理）
- 在步骤结束后点击「下一步」推进流程（也可开启自动推进）
- **必要时接管你的 AI 分身**：点击顶部右侧「替代分身」；顶部会显示**你的真实身份**（玩家视角下仅对自己可见）
- 需要操作时在消息区底部出现悬浮面板（发言 / 投票 / 狼队沟通与刀口），支持：
  - **快捷套话**：角色常用语 + 狼人杀术语 + 分身「思路」建议
  - **@ 提及**：输入 `@` 弹出在场玩家列表，插入 `@N号名字` 格式
  - **分身润色**：提交前由 AI 分身润色草稿，可选择发表润色版或原文
- 狼人夜间若你操控狼队角色：先队内私密沟通，再队内投票定刀口（平票随机），不进入公开频道
- 女巫夜间若你操控女巫角色：先决定是否对刀口使用解药，再选择毒药目标或不用（首夜不可毒）
- 守卫夜间若你操控守卫：选择守护目标（不可连续两晚守同一人）
- 预言家夜间若你操控预言家：选择查验目标（结果仅你可见）
- 通过 ESC 菜单暂停、调整音量、返回大厅
- 收听裁判 / 角色 TTS 播报（可在设置中开关；**玩家视角下跳过裁判解说 TTS**，仅保留角色发言；真人发言不强制 TTS）

**真人发言**在消息流中有独立样式标记（「真人发言」标签与粉紫气泡），与 AI 生成内容区分。

**对局记录**保存完整公开过程：参与角色、阶段时间线、发言、投票、胜负与消耗估算。进行中的对局可从中断处恢复；已结束的对局支持复盘，并可复制配置快速重开。

## 数据与素材

- **角色 / 玩法**：支持导入导出，便于备份与分享（不含对局记录与钱包）。内置玩法导出为 v2 配置包（含提示词与素材 manifest），可导入更新。
- **内置默认素材**：客户端自带 `default` 角色真实立绘/头像，以及 `custom` 玩法通用封面；未装完整包时，其他角色与各玩法封面会回退到上述默认素材。
- **完整素材包**：首次启动会尝试从 GitHub Release 下载；若失败可**跳过使用默认素材**，或**导入本地 zip**（设置 → 数据管理同样支持）。发版时在本地 `pnpm upload:release-assets` 上传（素材源为 `.dev-assets/`，不进 git）。
- **素材缺失回退**：未安装完整素材包，或某角色（如 doubao）缺少对应立绘/头像时，自动使用「默认」占位图，不阻断初始化与对局。
- **完整备份**：可在设置中心「数据管理」中操作本机数据。

生成内置占位素材（开发用）：

```bash
pnpm sync:bundled-assets
```

## 开发

```bash
pnpm install
cp .env.example .env   # 可选
pnpm dev
```

| 命令 | 说明 |
|------|------|
| `pnpm dev` | 开发模式 |
| `pnpm sync:bundled-assets` | 从 `.dev-assets` 同步 default 角色与 custom 玩法封面到 `bundled-assets/` |
| `pnpm pack:assets` | 从 `.dev-assets/` 打包 Release 用完整素材 zip |
| `pnpm upload:release-assets` | 本地打包素材并上传到 GitHub Release |
| `pnpm build:win` / `pnpm build:mac` | 本地打包安装包 |
| `pnpm push -- master` | 推送到 Gitee + GitHub（见 OPEN_SOURCE.md） |
| `pnpm typecheck` | 类型检查 |

技术栈：Electron + Vue 3 + TypeScript。开发约定见 [`AGENTS.md`](AGENTS.md)，构建见 [`docs/BUILD.md`](docs/BUILD.md)，开源发版见 [`docs/OPEN_SOURCE.md`](docs/OPEN_SOURCE.md)。
