# Agent Arena 使用说明

Agent Arena 是一款 Electron 桌面应用，用于创建 AI 角色、组织规则化社交对局，并通过 Platform 网关调用模型服务。

## 快速开始

1. 登录 Platform 账号并完成邮箱验证。
2. 首次启动完成素材与角色初始化后，**创建你的 AI 分身**。
3. 在大厅创建或选择角色，发起对局时可将分身选入席位。
4. 在设置中心的模型服务中确认网关与 Key 状态正常。

## 角色与对局

- **角色库**：每个角色绑定一个模型 ID，包含人设、说话风格与行为原则。
- **AI 分身**：全局唯一，代表「你」的替身；本局选中分身后，可在对局房间真人接管其发言。
- **玩法场景**：内置多种规则模板，决定身份分配、阶段流程与胜负条件。
- **对局记录**：本地保存对局进度，可在未完成时继续。

### 真人介入（接管分身）

适用条件：本局包含你的 AI 分身，且**尚未在本局切换过上帝视角**。

1. 进入对局房间，点击顶部右侧 **「替代分身」**（与视角切换、ESC 同一区域）。
2. 轮到你发言、投票或（狼人身份）夜间刀人时，**消息区底部**会出现悬浮输入面板。
3. 发言/沟通：在悬浮框输入内容，点击 **「确认提交」**（不限时）。
4. 投票/刀人：在悬浮框点选目标后确认；放逐投票可点 **「弃权」**。
5. 狼人夜间：先进行 **队内私密沟通**（仅狼队/上帝可见），再 **队内投票** 确认刀口；平票随机。
6. 可随时点 **「交还 AI」** 恢复 AI 操控。
7. 若已切换 **上帝视角**，本局将锁定真人接管，需新开一局才能再次介入。

默认进入房间为 **玩家视角**；上帝视角仅当局有效，下一局仍默认玩家视角。

## 模型服务

模型调用经 Platform 配置的 New API 网关：

- 在「模型概览」查看平台连接、网关地址与本机 Key。
- 在「连通性调试」浏览可用模型并执行 ping 测试。
- 在「流式调试」验证流式对话链路。
- **默认模型（兜底）**：在「模型概览」设置。角色绑定、玩法系统模型等**已选模型优先**；默认模型只在未配置或不可用时作为最终 fallback。帮助中心等没有独立模型选项的功能，会使用此兜底。

若模型无法回复，请依次检查：Platform 是否在线、邮箱是否验证、本机 Key 是否存在、钱包余额是否充足。

## 钱包与用量

- **钱包充值**：使用微信扫码充值，到账后可在用量统计中查看消费。
- **用量统计**：展示模型调用明细、Token 与费用趋势。
- **模型密钥**：管理 sk- 格式 API Key，对局与调试共用。

## 设置中心

- **用户信息**：查看个人资料与多维度数据统计。
- **应用设置**：调整主题、声音、对局默认与数据保留策略。
- **Agent 测评榜**：同步 Arena Agent Labs 厂商测评信号。

## 版本与更新

Agent Arena 目前处于**高速迭代期**——简单说就是：开发团队几乎每天都在发版，CHANGELOG 的滚动速度有时快过对局里的发言轮次。

### 更新有多勤？

- **几乎每天都有新内容**：界面微调、玩法细节、模型链路、帮助文档……总有一款在路上。
- 小改动可能悄悄上线；较大的功能会在「更新说明」里写明。
- 若你昨天用的按钮今天换了个位置，别慌，多半不是幻觉，是版本又刷新了。

### 谁在写这个项目？

坦白讲：是一群人类开发者，外加若干**被疯狂鞭策的 AI**。

典型工作流大致如下：

1. 人类：「这个交互再顺手一点。」
2. AI：「好的，已生成 800 行。」
3. 人类：「不对，重来，要更像人写的。」
4. AI：「……好的。」
5. 循环若干次，直到咖啡凉透或 CI 变绿。

所以你会看到一些文案既专业又略带生猛——那是**人审 + 机写**混合调制的风味，不是 bug，是特色（大概吧）。

### 更新后建议做什么？

1. 重启应用，确保加载最新构建。
2. 到设置中心查看「更新说明」，了解本次改了什么。
3. 若遇异常，先在「帮助中心」提问；文档里没有的，走「报 Bug」——开发同学（和它们的 AI 搭子）会跟进。

### 关于「稳定性」的诚实说明

我们追求可用，但不假装一切已完美封板。新功能可能还带着热气，旧界面可能正在迁移。感谢你在早期阶段愿意一起把 Arena 打磨得更好——你的反馈，会直接变成明天更新里的一条 commit 说明。

## 常见问题

### 页面显示不合适

进入「显示与动效」，调整界面缩放、主题模式与动画开关。

### 余额或用量异常

进入「钱包充值」与「用量统计」，核对最近订单与调用明细。

### 网关连接失败

确认 Platform 服务地址正确，并在「模型概览」中刷新连接状态。

### 如何提交 Bug

在「报 Bug」中填写问题描述，系统会保留完整历史记录供后续跟进。
